# Export PDF Statement

Feature สำหรับส่งออกรายการธุรกรรมเป็นไฟล์ PDF ในรูปแบบคล้ายใบยอดบัญชีของธนาคาร (bank statement) เพื่อนำไปแชร์หรือเก็บไว้ใช้

## 1. Current State (Updated)

- **Export:** **CSV** และ **PDF** (implemented)
- **API:** `GET /api/transactions/export` — คืน CSV (default) หรือ PDF เมื่อ `format=pdf`
- **UI:** Data Tools (`/dashboard/tools`), หน้ารายละเอียดบัญชี (`/dashboard/accounts/[id]`)
- **Filters:** from, to, type (INCOME/EXPENSE/TRANSFER), financialAccountId, locale (th/en)

## 2. Research: Bank Statement PDF Format

จากการวิจัยรูปแบบใบยอดบัญชีของธนาคาร (ไม่มี ISO standard — แต่ละธนาคารจัด layout ต่างกัน) โครงสร้างทั่วไปมี 3 ส่วนหลัก:

### 2.1 Header Section

- ชื่อธนาคาร/สถาบัน
- ชื่อเจ้าของบัญชี
- เลขบัญชี (มัก mask เป็น xxx-x-xxxxx-x)
- ประเภทบัญชี (checking, savings, etc.)
- ช่วงวันที่ของ statement (เช่น 1 มี.ค. – 31 มี.ค.)
- ที่อยู่ (optional)

### 2.2 Balance Summary

- ยอดยกมา (Opening balance)
- ยอดรวมฝาก/เครดิต (Total deposits & credits)
- ยอดรวมถอน/เดบิต (Total withdrawals & debits)
- ยอดปิดบัญชี (Closing balance)
- สูตร: Opening + Deposits − Withdrawals = Closing

### 2.3 Transaction Table (Body)

| คอลัมน์ | คำอธิบาย |
|--------|----------|
| Date | วันที่ทำธุรกรรม |
| Description | รายละเอียด/ร้านค้า |
| Reference | เลขอ้างอิง (optional) |
| Debit | ยอดเดบิต (ถอน/จ่าย) |
| Credit | ยอดเครดิต (ฝาก/รับ) |
| Balance | ยอดคงเหลือหลังรายการ |

- รายการเรียงตามวันที่ (มักใหม่สุดก่อน หรือเก่าสุดก่อน แล้วแต่ธนาคาร)
- บางแห่งมี running balance หลังแต่ละรายการ

### 2.4 Footer Section

- ค่าธรรมเนียม (maintenance, ATM, overdraft)
- ดอกเบี้ย (ถ้ามี)
- หมายเลขหน้า

## 3. Proposed PDF Statement Structure (Judtang)

ใช้โครงสร้างคล้ายธนาคาร แต่ปรับให้เหมาะกับข้อมูลที่มีในระบบ:

### 3.1 Header

| รายการ | แหล่งข้อมูล |
|--------|-------------|
| App name | "Judtang" (หรือจาก config) |
| ชื่อผู้ใช้ | User.name หรือ email |
| ชื่อบัญชี (ถ้า filter by account) | FinancialAccount.name |
| ประเภทบัญชี | BANK / CREDIT_CARD / WALLET / CASH / OTHER |
| ช่วงวันที่ | from – to (ตาม filter) |
| วันที่พิมพ์ | วันเวลาที่ export |

### 3.2 Balance Summary

| รายการ | การคำนวณ |
|--------|-----------|
| ยอดยกมา | ยอดคงเหลือต้นช่วง (จาก initialBalance + transactions ก่อน from) |
| รายรับรวม | SUM(amount) where type IN (INCOME, TRANSFER-in) |
| รายจ่ายรวม | SUM(amount) where type IN (EXPENSE, TRANSFER-out, PAYMENT) |
| ยอดปิดบัญชี | ยอดยกมา + รายรับ − รายจ่าย |

**หมายเหตุ:** ถ้า export ทั้งหมด (ไม่ filter by account) อาจใช้ "สรุปรวมทุกบัญชี" หรือแยกตามบัญชี

### 3.3 Transaction Table

| คอลัมน์ | แหล่งข้อมูล |
|---------|-------------|
| วันที่ | occurredAt (format ตาม locale) |
| ประเภท | INCOME / EXPENSE / TRANSFER / PAYMENT / INTEREST / ADJUSTMENT |
| รายละเอียด | category + note (หรือ note อย่างเดียว) |
| บัญชี (ถ้าไม่ filter) | financialAccount.name หรือ transferAccount.name |
| เดบิต | amount เมื่อ type = EXPENSE, TRANSFER-out, PAYMENT |
| เครดิต | amount เมื่อ type = INCOME, TRANSFER-in, INTEREST |
| ยอดคงเหลือ | running balance (ถ้ามี opening balance) |

### 3.4 Footer

- หมายเลขหน้า (Page X of Y)
- ข้อความ "Generated by Judtang" หรือ disclaimer

## 4. Implementation Approach

### 4.1 Library Options

| Library | Pros | Cons |
|---------|------|------|
| **@react-pdf/renderer** | React components, declarative, pagination built-in | ต้องใช้ใน Node/Edge context สำหรับ server-side |
| **pdf-lib** | Low-level, ไม่ต้องใช้ React, ควบคุม layout ได้ละเอียด | ต้องจัดการ pagination เอง |

**แนะนำ:** `@react-pdf/renderer` — เหมาะกับโครงสร้าง document ที่มี header/table/footer ชัดเจน และใช้ React ได้

### 4.2 API Design

**Option A: Query param ต่อจาก export เดิม**

```
GET /api/transactions/export?format=pdf&from=...&to=...&type=...&financialAccountId=...
```

- `format=csv` (default) → CSV
- `format=pdf` → PDF

**Option B: Endpoint แยก**

```
GET /api/transactions/export/pdf?from=...&to=...&...
```

**แนะนำ:** Option A — ใช้ endpoint เดียว แยก logic ตาม `format`

### 4.3 Response

- `Content-Type: application/pdf`
- `Content-Disposition: attachment; filename="statement-YYYYMMDD.pdf"`
- Activity Log: `TRANSACTION_EXPORT` พร้อม `format: "pdf"`

### 4.4 UI

- Data Tools: เพิ่มปุ่ม "ดาวน์โหลด PDF" หรือ dropdown "CSV | PDF"
- หน้ารายละเอียดบัญชี: ปุ่ม export PDF (ใช้ financialAccountId จาก context)
- ใช้ filter เดียวกับ CSV (date range, type)

### 4.5 Data Requirements

- Transaction ต้อง include: `financialAccount`, `transferAccount` (สำหรับชื่อบัญชี)
- User: `name`, `email`
- FinancialAccount (เมื่อ filter): `name`, `type`, `initialBalance`
- ต้องคำนวณ opening/closing balance ตามช่วงวันที่

## 5. Edge Cases

- **ไม่มี transaction ในช่วง:** แสดงเฉพาะ header + summary (ยอดยกมา = ยอดปิด)
- **หลายหน้า:** ต้อง repeat header บนทุกหน้า (หรืออย่างน้อยหน้าแรก)
- **หลายบัญชี (ไม่ filter):** แสดงชื่อบัญชีในคอลัมน์ หรือแยก section ตามบัญชี
- **TRANSFER:** แสดงทั้งต้นทางและปลายทาง หรือแสดงแค่ฝั่งที่เกี่ยวข้องเมื่อ filter by account

## 6. Out of Scope (Phase 1)

- Watermark / password protection
- Custom branding ต่อบัญชี
- Export เป็นหลายไฟล์ (แยกบัญชี)
